# Kura-Tas プロジェクト仕様書 (spec.md)

> [!IMPORTANT]
> 本資料は、利用者が「何ができるか（What）」に焦点を当てた機能仕様書です。技術的な実装詳細（How）については、[architecture.md](../architecture.md) を参照してください。

## 1. プロジェクト概要
HTML 1ファイルで完結し、File System Access API を通じてローカルの JSON/Markdown ファイルを操作する、エンジニア向けの爆速タスク管理・ナレッジハブ。

## 2. コア・コンセプト
- **No-Server, No-Cloud**: 全てのデータはボスの手元（ローカルフォルダ）に。
- **Speed Over Everything**: マウス移動とクリック回数を最小化し、キーボードショートカットをフル活用する。
- **Knowledge-Driven**: タスクと日付（Daily）の両方に紐づく手順書（Markdown）をナレッジの核とする。

## 3. 主要機能 (Core Features)

### 3.1. タスク管理
- **デイリータスク**: 日付ごとにタスクを管理。完了/未完了の切り替えが可能。
- **他者依頼（Delegated）マーク**: 右クリックメニューからトグル可能。自分ではなく他者にボールがあるタスクを視覚的に区別（左端に赤いボーダー）できる。
- **繰り越し (Carry-over)**: 過去の未完了タスクをワンクリックで今日の日付へ一括移動できる。
- **インポート**: AI等の外部ツールで生成されたJSON形式のタスクリスト（JSON配列）や、単純な改行区切りのテキスト（1行1タスク）を一括で読み込める。
  - **入力形式**: 「自動判別」「JSON形式」「改行区切りテキスト」から選択可能。
  - **自動判別**: 入力テキストが有効なJSON配列としてパースできる場合はJSONインポートを行い、それ以外（パースエラーや配列でない場合）は改行区切りテキストとして処理する。
  - **空行のハンドリング**: 改行区切りテキストのインポート時、空行（トリムして空になる行）は無視される。

### 3.2. ナレッジ・コンテキスト (Notes)
- **デイリーメモ**: その日全体の備忘録（Markdown）。
  - ノート表示時のタイトルには、日付が表示される（例: `Daily: 2026-07-09`）。
- **タスクメモ**: 個別のタスクに紐づく手順書やログ（Markdown）。
  - ノート表示時のタイトルには、紐づくタスクのテキストが表示される（例: `Task: タスク名`）。紐づくタスクが見つからない場合はフォールバックとして `Task: 無題のノート` と表示される。
- **共通リンク集**: 全ての日付から参照可能なブックマーク機能。

### 3.3. 自動化と効率化
- **定期タスク**: スケジュール自動生成ルールを設定可能。
  - **スケジュールタイプ**:
    - **毎週 (Weekly)**: 指定した曜日に毎週生成。
    - **週おき (Interval)**: 指定した基準日から N 週間ごとの指定曜日に生成。
    - **毎月日付 (Monthly Day)**: 毎月指定日（1〜31日、または「月末」）に生成。指定日がその月に存在しない（例：30日までの月に31日を指定）場合は、その月の最終日に前倒しして生成される。
    - **毎月第N曜日 (Monthly Weekday)**: 毎月の第1〜第5、または最終の指定曜日に生成（例：第2火曜日、最終金曜日）。
  - **自動生成**: 当日および未来の日付に自動でタスクが追加される。
  - **同期 (Sync)**: マスタの内容（テキスト）を変更すると、既に生成済みの当日・未来のタスクにも変更が反映される。
  - **ノートテンプレート自動生成と同期**:
    - 定期タスクマスタにノートテンプレート（Markdown本文）を保持可能。
    - 定期タスクからタスクが生成（自動・手動）される際、テンプレートが存在すれば、そのタスク専用のノートが自動作成される。
    - **テンプレートへの昇格**: 日々のタスクノートUIから「👑 テンプレートに昇格」を実行することで、現在のノート本文を定期タスクのテンプレートとして登録・更新できる。
    - **安全な同期**: テンプレート更新（昇格含む）時に、当日・未来の「未完了」かつ「未編集（本文が空、または更新前のテンプレートと同一）」のタスクノートのみに新テンプレートを自動上書き同期し、編集済みノートは保護する。
    - **通常タスクからの登録と同期**: 通常タスクの右クリックメニューから「定期タスクに登録」を選択することで、そのタスクの名前と紐づくノート（メモ）の本文を引き継いだ状態で定期タスク登録モーダルを開き、マスタを新規作成できます。
  - **実績・手動操作の保護**: 
    - 完了済みのタスクは削除されない。
    - ユーザーが手動で別の日付に移動させたタスクは、スケジュールの変更（曜日の廃止）による自動削除の対象外となり、内容の更新のみが行われる。
  - **休日制御**:
    - アプリ設定で定義された営業日の曜日（デフォルト: 月〜金）および特定の祝日・休日の日付リスト（YYYY-MM-DD形式）に基づいて休日を自動判定します。
    - 定期タスクごとに「休日調整ルール」（スキップ / 前営業日に移動 / 後営業日に移動）を設定できます。
    - 重複調整により移動されたタスクは、本来の予定日を内部データ（originalDate）に保持し、自動削除ロジック（手動操作の保護）と正しく連携します。
- **爆速オペレーション**: 
  - マウスの右クリック（コンテキストメニュー）や中クリック（即時完了/移動）による高速操作。
  - 編集 ⇔ プレビューのトグルや保存、Undo/Redo を網羅したキーボードショートカット。

### 3.4. 状態の永続化と復元 (Persistence & History)
- **自動保存**: メモの編集や操作は即座にローカルファイルへ反映される。
- **Undo/Redo**: 
  - 最大30件（デフォルト、Configで変更可能）の操作履歴を保持。
  - メインツールバーの Undo / Redo ボタン、またはショートカットキー（Ctrl+Z / Ctrl+Y）で操作可能。
  - 履歴スタックはアプリケーションの状態全体のスナップショットとして保持される。
- **ディレクトリ再開 (Resume)**: 最後に使用したディレクトリを記憶し、起動時にスムーズに再開できる。

### 3.5. バージョン情報 (Version Info)
- **バージョン表記の一元化**: `package.json` の `version` を正とし、ビルド時に自動で画面のバージョン情報やブラウザのタイトルに反映する。
- **デバッグ情報**: デバッグモードが有効な場合、バージョン表記の横に Git コミットハッシュおよびビルド日時（JST）を表示する（例: `v1.0.0 (commit: abc123f, built: 2026/07/16 18:00)`）。

## 4. 画面構成 (UI Layout)
### A. 左サイドバー (Navigation & Inbox)
- **Navigation**: 今日への移動、カレンダー（WIP）、定期タスク管理、タスク読込（AI連携）。
- **Inbox (Stock)**: 走り書きメモの一時保管場所。クリックでタスク化。

### B. 中央メイン (Daily Task List)
- 日付ナビゲーションと繰り越しツールバー。
- タスク一覧（ステータス表示、選択によるノート切り替え）。

### C. 右パネル (Knowledge Context)
- Markdownエディタ/プレビュー。
- 共通リンク集管理エリア。

## 6. データ構造 (Data Structures)
### ローカルファイル
- `/tasks/[YYYY-MM-DD].json`: 日付別タスクデータ。
- `/inbox.json`: Inboxデータ。
- `/links.json`: 共通リンク集。
- `/routine.json`: 定期タスク定義。
  ```typescript
  export interface RoutineTask {
    id: string;
    text: string;
    schedule: {
      type: 'weekly' | 'interval' | 'monthly-day' | 'monthly-weekday' | 'none';
      days?: DayOfWeekStr[];         // 'weekly', 'interval', 'monthly-weekday' 用
      intervalWeeks?: number;        // 'interval' 用
      baseDate?: string;             // 'interval' 用
      monthlyDay?: number | 'last';  // 'monthly-day' 用
      weekIndex?: number | 'last';   // 'monthly-weekday' 用
    };
    lastGenerated?: string;
    holiday_adjustment?: 'before' | 'after' | 'skip';
  }
  ```
- `/config.json`: アプリ設定。
- `/notes/*.md`: 手順書（フロントマター付きMarkdown）。

### ブラウザ内キャッシュ
- **IndexedDB**: ディレクトリハンドルの保持に使用。
