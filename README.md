# Kura-Tas

エンジニアによる、エンジニアのための、爆速ナレッジ融合型htmlタスク管理ツール。

## 🚀 プロジェクト概要
Kura-Tas は、HTML 1ファイルで完結する超軽量なタスク管理・ナレッジハブです。
ブラウザの **File System Access API** を活用し、データはすべて利用者のローカルディレクトリに JSON/Markdown 形式で保存されます。サーバー不要、クラウド不要、プライバシーとスピードを最優先に設計されています。

## 🎯 ターゲット層
- 既存のタスク管理ツール（Excel, Notion, Jira等）の「入力の重さ」や「動作の鈍さ」に疲弊しているエンジニア。
- タスク実行時に「手順書はどこだっけ？」「関連リンクは何だっけ？」というスイッチングコストをゼロにしたいマルチタスカー。
- 会社の制約などで外部SaaSが使いにくいが、手書きメモ以上の効率を求めるプロフェッショナル。

## 💎 既存ツールとの決定的な違い
1. **ナレッジとの不可分性**: タスクと手順書（Markdown）、デイリーログが三位一体で管理されます。
2. **爆速オペレーション**: マウスのホイールクリック（完了トグル）や `Ctrl+P`（プレビュー）など、思考の速度を落とさない操作系。
3. **1ファイル完結のポータビリティ**: `src/index.html` 1つをどこへ持っていっても、ブラウザがあれば即座に自分専用の作業環境が再現されます。
4. **Sakura Modernism**: エンジニアに馴染み深い「サクラエディタ」の配色を纏い、長時間の集中をサポートします。

## 📖 使い方
1. `src/index.html` を Chrome または Edge で開きます。
2. 「ディレクトリを開く」ボタンを押し、タスク・メモを保存したいローカルフォルダを選択します。
3. **Inbox**: 思いついたことを走り書きし、クリックでその日のタスクに昇格させます。
4. **Daily Note**: 日付をクリックして、その日全体の備忘録を Markdown で記録します。
5. **Task Note**: タスクを選択し、右パネルでそのタスク専用の手順書を記述・閲覧します。

## 🛠 技術スタック
- **Frontend**: Vanilla HTML / CSS / JS (ES6+)
- **Storage**: Web File System Access API / IndexedDB
- **Parser**: 自作軽量 Markdown Parser (No Dependencies)

## 🛠 開発者向けガイド

### 環境構築
以下でdist/index.htmlに成果物が作成されます。
```
npm install
npm run build
```

### ビルド手順
本プロジェクトは Vite を使用してビルドされます。

- **通常ビルド**: `npm run build`
- **デバッグビルド**: `npm run build:debug` (デバッグモードが最初からONになります)

> [!IMPORTANT]
> **バージョンの整合性について**
> ビルドされる HTML には最新の Git コミットハッシュが埋め込まれます。
> 修正内容を正確にバージョン情報に反映させるため、**必ず `git commit` を行った後にビルドを実行してください。**

### デバッグモード
デバッグモードを有効にすると、コンソールに詳細な操作ログ（保存、読込、レンダリング等）が出力され、セットアップ画面にバージョン情報が表示されます。

- **有効化方法**:
  - `npm run build:debug` でビルドしたファイルを使用する。
  - 通常ビルド版でも、URLに `?debug=1` パラメータを付与してアクセスする。
